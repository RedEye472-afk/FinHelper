# 🧮 Математическое ядро «ФинПомощник»

Этот документ содержит все математические формулы, которые использует приложение. Каждая формула имеет:

- Числовой пример

**Важно**: Все денежные расчёты выполняются на `decimal(28)`. Запрещено использовать `float` для денег.

## 💰 1. TVM (Time Value of Money) — Копнова Гл. 1-2

### 1.1. Простые проценты

**Формула**:
```
S = P × (1 + i × t)
```

**Переменные**:
- `S` — итоговая сумма (наращенная сумма)
- `P` — principal (начальная сумма)
- `i` — годовая процентная ставка (в долях, например 0.10 = 10%)
- `t` — время в годах

**Пример**:
- P = 100 000₽, i = 10% (0.10), t = 0.5 года (6 месяцев)
- S = 100 000 × (1 + 0.10 × 0.5) = 100 000 × 1.05 = **105 000₽**

**Где используется**: Краткосрочные вклады (< 1 года), расчеты между контрагентами

**Edge-cases**:
- `t = 0` → S = P (нет начислений)
- `i = 0` → S = P (беспроцентный период)
- `t < 0` → ошибка (время не может быть отрицательным)

---

### 1.2. Сложные проценты (с капитализацией)

**Формула**:
```
S = P × (1 + i/m)^(m×t)
```

**Переменные**:
- `S` — итоговая сумма
- `P` — начальная сумма
- `i` — годовая процентная ставка
- `m` — частота капитализации в год (12 = ежемесячно, 4 = ежеквартально, 1 = ежегодно)
- `t` — срок в годах

**Пример**:
- P = 100 000₽, i = 10% (0.10), m = 12 (ежемесячно), t = 1 год
- S = 100 000 × (1 + 0.10/12)^(12×1) = 100 000 × (1.00833)^12 = **110 471.31₽**

**Где используется**: Вклады с капитализацией, долгосрочные инвестиции

**Edge-cases**:
- `m = 1` → формула упрощается до S = P × (1 + i)^t
- `t = 0` → S = P
- `i = 0` → S = P

---

### 1.3. Эффективная годовая ставка

**Формула**:
```
i_eff = (1 + i_nom/m)^m − 1
```

**Переменные**:
- `i_eff` — эффективная годовая ставка
- `i_nom` — номинальная годовая ставка
- `m` — частота капитализации

**Пример**:
- i_nom = 10% (0.10), m = 12 (ежемесячно)
- i_eff = (1 + 0.10/12)^12 − 1 = 1.10471 − 1 = **0.10471 (10.471%)**

**Где используется**: Сравнение вкладов с разной частотой капитализации

**Зачем это нужно**: Вклад "10% с ежемесячной капитализацией" на самом деле даёт 10.47% годовых. Без этой формулы нельзя честно сравнить два вклада.

---

### 1.4. Формула Фишера (реальная доходность)

**Формула**:
```
r_real = (1 + r_nom) / (1 + π) − 1
```

**Переменные**:
- `r_real` — реальная доходность (после инфляции)
- `r_nom` — номинальная доходность
- `π` — инфляция (в долях)

**Пример**:
- r_nom = 10% (0.10), π = 8% (0.08)
- r_real = (1 + 0.10) / (1 + 0.08) − 1 = 1.10 / 1.08 − 1 = **0.01852 (1.852%)**

**Где используется**: Калькулятор вкладов (показать реальную доходность после инфляции)

**Упрощённая формула** (при малых значениях):
```
r_real ≈ r_nom − π
```
- r_real ≈ 10% − 8% = 2% (приблизительно)

**Edge-cases**:
- `π = -1` → деление на ноль (дефляция 100% невозможна)
- `π > r_nom` → реальная доходность отрицательная (деньги обесцениваются)

## 🏦 2. Кредиты — Копнова Гл. 4

### 2.1. Аннуитетный платёж (равные платежи)

**Формула**:
```
A = P × [i × (1+i)^n] / [(1+i)^n − 1]
```

**Переменные**:
- `A` — ежемесячный платёж (аннуитет)
- `P` — сумма кредита (principal)
- `i` — месячная процентная ставка (годовая / 12)
- `n` — количество месяцев (срок кредита)

**Пример**:
- P = 1 000 000₽, годовая ставка = 12%, n = 24 месяца
- i = 0.12 / 12 = 0.01 (1% в месяц)
- A = 1 000 000 × [0.01 × (1.01)^24] / [(1.01)^24 − 1]
- A = 1 000 000 × [0.01 × 1.26973465] / [1.26973465 − 1]
- A = 1 000 000 × 0.01269735 / 0.26973465
- A = **47 073.47₽** (точное значение 47 073.4722232…, округление HALF_EVEN;
  сверка: Excel ПЛТ(0.12/12; 24; -1000000) = -47073.472223…)

**Где используется**: Кредитный калькулятор, ипотека

**Fallback при i = 0**:
```
A = P / n
```
- Если ставка 0%, платёж = сумма / количество месяцев

**Edge-cases**:
- `i = 0` → использовать fallback A = P / n
- `n = 0` → ошибка (срок не может быть 0)
- `P = 0` → A = 0 (нет кредита)
- `n < 0` → ошибка

---

### 2.2. Дифференцированный платёж (убывающие платежи)

**Формула для k-го месяца**:
```
Платёж_k = P/n + (P − Σ_{j=1}^{k-1} P/n) × i
```

**Переменные**:
- `Платёж_k` — платёж в k-й месяц
- `P` — сумма кредита
- `n` — количество месяцев
- `i` — месячная ставка
- `k` — номер месяца (от 1 до n)

**Пример** (первые 2 месяца):
- P = 1 000 000₽, i = 1% (0.01), n = 24 месяца
- Ежемесячное погашение долга: P/n = 1 000 000 / 24 = 41 666.67₽

**Месяц 1**:
- Платёж_1 = 41 666.67 + (1 000 000 − 0) × 0.01 = 41 666.67 + 10 000 = **51 666.67₽**

**Месяц 2**:
- Платёж_2 = 41 666.67 + (1 000 000 − 41 666.67) × 0.01 = 41 666.67 + 9 583.33 = **51 250.00₽**

**Где используется**: Кредитный калькулятор (альтернатива аннуитету)

**Особенность**: Первый платёж самый большой, последний — самый маленький. Общая переплата меньше, чем при аннуитете.

**Edge-cases**:
- `i = 0` → все платежи равны P/n
- `n = 0` → ошибка
- `k > n` → ошибка (месяц вне срока)

---

### 2.3. Полная стоимость кредита (ПСК)

**Определение** (по методике ЦБ РФ, Указание 5750-У от 31.05.2024, ранее 5832-У):
ПСК — это **годовая процентная ставка**, при которой приведённая стоимость всех денежных потоков по кредиту равна нулю. Это внутренняя норма доходности (IRR) денежного потока с точки зрения заёмщика.

**Уравнение для численного решения** (метод Ньютона / BrentQ):
```
Σ_{k=0}^{n}  CF_k / (1 + i)^((d_k − d_0) / 365) = 0
```

**Переменные**:
- `CF_k` — денежный поток в дату `d_k` (положительный = получение денег заёмщиком, отрицательный = оплата)
- `d_k` — дата k-го потока; `d_0` — дата первого потока (база)
- `i` — годовая ПСК (решение уравнения)
- DayCount: ACT/365 (по умолчанию для потребительских кредитов РФ)

**Что включает ПСК** (CF с точки зрения заёмщика):
- Тело полученного кредита (CF_0 > 0)
- Все проценты по кредиту (CF_k < 0)
- Обязательные страховки (CF_k < 0)
- Комиссии за выдачу/обслуживание (CF_k < 0)
- Плата за оценку залога, если обязательна (CF_k < 0)

**Что НЕ включает ПСК**:
- Добровольные страховки
- Штрафы и пени
- Плата за досрочное погашение

**Пример** (с ежемесячными платежами, ЧБП = 12):
- P = 1 000 000₽, срок 24 месяца, номинальная ставка 12%
- Страховка: 50 000₽ (единоразово, в дату выдачи)
- Комиссия: 5 000₽ (единоразово, в дату выдачи)
- Аннуитетный платёж: 47 073.47₽ (точное 47 073.4722232…)

**Денежные потоки** (заёмщик):
- CF_0 = +1 000 000 − 50 000 − 5 000 = **+945 000**
- CF_1..CF_24 = **−47 073.4722232…** каждый месяц

**Решение уравнения** (численно, BrentQ; ставка за базовый период i, ПСК = i × ЧБП):
- `945 000 = Σ_{k=1}^{24} 47 073.4722… / (1 + i)^q_k`, где `q_k = (d_k − d_0) × 12 / 365`
- Ставка за период ≈ 1.480%, ПСК ≈ **17.76%** (номинальная годовая по методике ЦБ)

> **Важно для реализации**: точное значение ПСК получается только численным решением.
> Упрощённые формулы вида `(Σ платежей − P) / P × (12/n)` дают **грубую апроксимацию**
> и не учитывают временную стоимость денег — не использовать.
>
> **Методология ЦБ (Указание 5750-У)**: ПСК — это **номинальная** годовая
> ставка `i × ЧБП`, а не эффективная `(1+i)^ЧБП − 1`. Кредит 12% годовых без
> доп. платежей даёт ПСК ровно 12%, совпадая с рекламируемой ставкой.
> Эффективная ставка при этом ~12.68% — это другая метрика, её ПСК не равна.

**Где используется**: Кредитный калькулятор (показать реальную стоимость кредита)

**Зачем это нужно**: Банк рекламирует "12% годовых", но с учётом обязательной страховки и комиссии реальная стоимость кредита — около 18% годовых. ПСК показывает честную картину.

**Edge-cases**:
- Нет дополнительных платежей → ПСК = номинальная ставка
- Все доп. платежи добровольные → ПСК = номинальная ставка
- Все CF одного знака → ПСК не определена (нет решения)
- Несколько смен знака в CF → мульти-корни, брать наименьший положительный

---

### 2.4. Досрочное погашение

**Пересчёт остатка**:
```
Остаток_k = P − Σ_{j=1}^{k} (основной долг_j)
```

**Два варианта досрочки**:
1. **Сокращение срока**: платёж остаётся тем же, срок уменьшается
2. **Сокращение платежа**: срок остаётся тем же, платёж уменьшается

**Экономия на процентах**:
```
Экономия = Σ процентов_до − Σ процентов_после
```

**Пример** (сокращение срока):
- Кредит 1 000 000₽, 12%, 24 месяца, платёж 47 073.46₽
- После 12 месяцев остаток: 523 000₽
- Досрочное погашение: 100 000₽
- Новый остаток: 423 000₽
- Новый срок: ~18 месяцев (вместо оставшихся 12)
- Экономия: ~15 000₽ на процентах

**Где используется**: Кредитный калькулятор (сценарий досрочки)

**Edge-cases**:
- Досрочка = 0 → нет изменений
- Досрочка ≥ остаток → кредит закрыт
- Досрочка < ежемесячного платежа → ошибка

## 📈 3. Инвестиции — Копнова Гл. 6

### 3.1. NPV (Net Present Value, чистая приведённая стоимость)

**Формула**:
```
NPV = Σ CF_t / (1+r)^t
```

**Переменные**:
- `NPV` — чистая приведённая стоимость
- `CF_t` — денежный поток в период t (отрицательный = инвестиция, положительный = доход)
- `r` — ставка дисконтирования
- `t` — период (год, месяц)

**Пример**:
- Инвестиция: -100 000₽ (сегодня, t=0)
- Доходы: 30 000₽ (год 1), 40 000₽ (год 2), 50 000₽ (год 3)
- Ставка дисконтирования: 10% (0.10)
- NPV = -100 000 + 30 000/(1.10)^1 + 40 000/(1.10)^2 + 50 000/(1.10)^3
- NPV = -100 000 + 27 272.73 + 33 057.85 + 37 565.74
- NPV = **-2 103.68₽** (проект убыточный)

**Где используется**: Оценка инвестиционных проектов

**Правило принятия решения**:
- NPV > 0 → проект прибыльный, принимать
- NPV < 0 → проект убыточный, отклонять
- NPV = 0 → проект безубыточный

**Edge-cases**:
- Все CF отрицательные → NPV < 0 (только инвестиции, нет доходов)
- r = 0 → NPV = Σ CF_t (без дисконтирования)
- r < 0 → ошибка (отрицательная ставка дисконтирования невозможна)

---

### 3.2. XIRR (Extended Internal Rate of Return)

**Формула** (уравнение для нахождения r):
```
Σ CF_t / (1+r)^(days_t/365) = 0
```

**Метод решения**: BrentQ с bracket `[-0.99, 10.0]`

**Переменные**:
- `CF_t` — денежный поток в дату t
- `days_t` — количество дней от начальной даты до даты t
- `r` — годовая доходность (XIRR)

**Пример**:
- 01.01.2024: -100 000₽ (инвестиция)
- 01.04.2024: +30 000₽ (доход)
- 01.07.2024: +40 000₽ (доход)
- 01.10.2024: +50 000₽ (доход)
- XIRR = **40.66%** (решается численно, ACT/365; проверка: NPV при этой ставке ≈ 0)

> **Важно**: XIRR — это **эффективная** годовая доходность (через экспоненту
> days/365), в отличие от ПСК (§2.3), которая считается как номинальная
> годовая по методике ЦБ. Не путать метрики.

**Где используется**: Реальная доходность портфеля с нерегулярными денежными потоками

**Fallback при мульти-корнях**:
Если уравнение имеет несколько решений → использовать MIRR

**Edge-cases**:
- Все CF одного знака → XIRR не определён (нет смены знака)
- Только один CF → XIRR не определён
- Нет смены знака (все отрицательные или все положительные) → ошибка

---

### 3.3. MIRR (Modified Internal Rate of Return)

**Формула**:
```
MIRR = (FV_positive_CF / PV_negative_CF)^(1/n) − 1
```

**Переменные**:
- `FV_positive_CF` — будущая стоимость положительных CF (реинвестированных)
- `PV_negative_CF` — приведённая стоимость отрицательных CF
- `n` — количество периодов

**Пример**:
- Отрицательные CF: -100 000₽ (t=0)
- Положительные CF: 30 000₽ (t=1), 40 000₽ (t=2), 50 000₽ (t=3)
- Ставка реинвестирования: 8%
- Ставка финансирования: 10%
- PV_negative_CF = 100 000₽
- FV_positive_CF = 30 000×(1.08)^2 + 40 000×(1.08)^1 + 50 000 = 34 992 + 43 200 + 50 000 = 128 192₽
- MIRR = (128 192 / 100 000)^(1/3) − 1 = **8.63%** (кубический корень из 1.28192 = 1.08631; ранее в тексте было округление 8.52% — арифметическая ошибка)

**Где используется**: Альтернатива XIRR при мульти-корнях

**Edge-cases**:
- Нет положительных CF → MIRR не определён
- Нет отрицательных CF → MIRR не определён

---

### 3.4. DPP (Discounted Payback Period, дисконтированный срок окупаемости)

**Формула**:
```
DPP = период, когда Σ discounted_CF ≥ 0
```

**Пример**:
- Инвестиция: -100 000₽ (t=0)
- Доходы: 30 000₽ (год 1), 40 000₽ (год 2), 50 000₽ (год 3)
- Ставка: 10%
- Дисконтированные CF: -100 000, 27 272.73, 33 057.85, 37 565.74
- Кумулятивно: -100 000, -72 727.27, -39 669.42, -2 103.68
- DPP > 3 лет (не окупается за 3 года)

**Где используется**: Оценка инвестиционных проектов

**Edge-cases**:
- Проект не окупается → DPP = ∞ (бесконечность)
- Окупается в середине периода → интерполяция

---

### 3.5. PI (Profitability Index, индекс прибыльности)

**Формула**:
```
PI = PV_future_CF / initial_investment
```

**Переменные**:
- `PV_future_CF` — приведённая стоимость будущих денежных потоков
- `initial_investment` — начальные инвестиции

**Пример**:
- Инвестиция: 100 000₽
- PV будущих CF: 115 000₽
- PI = 115 000 / 100 000 = **1.15**

**Правило принятия решения**:
- PI > 1 → проект прибыльный
- PI < 1 → проект убыточный
- PI = 1 → проект безубыточный

**Где используется**: Оценка инвестиционных проектов

**Edge-cases**:
- initial_investment = 0 → PI не определён (деление на ноль)
- initial_investment < 0 → ошибка (инвестиции не могут быть отрицательными)

## 📅 4. DayCount Conventions

### 4.1. ACT/365 (фактические дни / 365)

**Формула**:
```
year_fraction = (end_date − start_date).days / 365
```

**Пример**:
- start_date = 2024-01-01
- end_date = 2024-07-01
- days = 182
- year_fraction = 182 / 365 = **0.49863**

**Где используется**: Вклады в РФ (по умолчанию)

---

### 4.2. 30/360 ISDA

**Формула**:
```
days = 360 × (Y2 − Y1) + 30 × (M2 − M1) + (D2 − D1)
year_fraction = days / 360
```

**Правила корректировки**:
- Если D1 = 31 → D1 = 30
- Если D2 = 31 и D1 ≥ 30 → D2 = 30

**Пример**:
- start_date = 2024-01-31
- end_date = 2024-03-31
- После корректировки: D1 = 30, D2 = 30
- days = 360×(2024−2024) + 30×(3−1) + (30−30) = 60
- year_fraction = 60 / 360 = **0.16667**

**Где используется**: Облигации, корпоративные кредиты

---

### 4.3. ACT/ACT ISMA

**Формула**:
```
year_fraction = Σ (days_in_year_i / days_in_year_i)
```

**Особенность**: Для високосных годов учитывает фактическое количество дней (366 вместо 365)

**Пример**:
- start_date = 2024-01-01 (високосный год)
- end_date = 2024-07-01
- days = 182
- year_fraction = 182 / 366 = **0.49727**

**Где используется**: Государственные облигации (ОФЗ)

---

### 4.4. Абстракция DayCountEngine

**Интерфейс**:
```
year_fraction(start: date, end: date, convention: str) → Decimal
```

**Поддерживаемые конвенции**:
- `ACT/365` (default для РФ)
- `30/360` (ISDA)
- `ACT/ACT` (ISMA)

**Edge-cases**:
- `start_date = end_date` → year_fraction = 0
- `start_date > end_date` → отрицательный year_fraction (ошибка или предупреждение)
- Неизвестная конвенция → ошибка

## 💸 5. Налоги РФ (versioned in configs/)

### 5.1. НПД (Налог на профессиональный доход)

**Формула**:
```
Налог = доход × ставка
```

**Ставки**:
- 4% — при работе с физлицами
- 6% — при работе с ИП и юрлицами

**Лимит**: 2.4 млн ₽/год

**Пример**:
- Доход от физлиц: 500 000₽
- Доход от ИП: 300 000₽
- Налог = 500 000 × 0.04 + 300 000 × 0.06 = 20 000 + 18 000 = **38 000₽**

**Edge-cases**:
- Доход > 2.4 млн ₽ → переход на УСН (не НПД)
- Доход = 0 → налог = 0
- Отрицательный доход → ошибка

---

### 5.2. УСН (Упрощённая система налогообложения)

**Формулы**:
```
УСН 6%: Налог = доход × 0.06
УСН 15%: Налог = (доход − расходы) × 0.15
```

**Deductible expenses** (для УСН 15%):
- Зарплата сотрудников
- Аренда
- Материалы
- Страховые взносы

**Пример** (УСН 15%):
- Доход: 2 000 000₽
- Расходы: 1 200 000₽
- Налог = (2 000 000 − 1 200 000) × 0.15 = 800 000 × 0.15 = **120 000₽**

**Edge-cases**:
- Расходы > доходы → налог = 0 (но не возврат)
- Доход = 0 → налог = 0

---

### 5.3. НДФЛ (Налог на доходы физических лиц)

**Формула**:
```
НДФЛ = (доход − вычеты) × 0.13
```

**Стандартные вычеты**:
- 1 400₽/мес на 1 ребёнка
- 3 000₽/мес на 3+ детей

**Имущественный вычет**:
- До 260 000₽ (13% от 2 млн ₽) — при покупке жилья

**Инвестиционный вычет (ИИС тип А)**:
- До 52 000₽/год (13% от 400 000₽)

**Пример**:
- Доход: 1 000 000₽
- Вычет на 1 ребёнка: 1 400 × 12 = 16 800₽
- НДФЛ = (1 000 000 − 16 800) × 0.13 = 983 200 × 0.13 = **127 816₽**

**Edge-cases**:
- Доход < вычеты → НДФЛ = 0 (но не возврат)
- Доход = 0 → НДФЛ = 0

---

### 5.4. Налог на вклад (ФЗ-382 от 26.12.2023, действует с доходов 2024+)

**Важно**: С налогового периода 2024 года (ФЗ-382 от 26.12.2023) методика расчёта
изменилась. **Не использовать устаревшую формулу "максимальная ставка за год"**,
которая действовала для доходов 2021-2023.

**Формула** (актуальная для 2024-2026):
```
Налог = max(0, доход_по_вкладам − 1_000_000 × ключевая_ставка_ЦБ_на_1_января) × 0.13
```

**Порог**: 1 млн ₽ × **ключевая ставка ЦБ на 1 января** года, за который считается доход.

**Пример** (доходы 2024 года):
- Ключевая ставка ЦБ на 01.01.2024: 16%
- Порог: 1 000 000 × 0.16 = **160 000₽**
- Доход по вкладам за 2024 год: 200 000₽
- Налог = max(0, 200 000 − 160 000) × 0.13 = 40 000 × 0.13 = **5 200₽**

**Пример** (доходы 2026 года):
- Ключевая ставка ЦБ на 01.01.2026: подставляется из справочника (`configs/tax_rules_2026.yaml`)
- Если, например, ставка = 21%: порог = 1 000 000 × 0.21 = 210 000₽
- При доходе 300 000₽: налог = (300 000 − 210 000) × 0.13 = 11 700₽

**Где используется**: Калькулятор вкладов

**Версионирование**: Ставка ЦБ на 1 января и ставка НДФЛ меняются по годам →
хранить в `configs/tax_rules_{год}.yaml`. Каждое поле имеет `source` ссылку.

**Edge-cases**:
- Доход ≤ порога → налог = 0
- Доход = порогу → налог = 0
- Ключевая ставка на 1 января = 0 → порог = 0, налог = доход × 0.13 (гипотетический случай)
- В 2021-2023 применялась формула "макс ставка за год" — для обратной совместимости
  реализовать как отдельный вариант в `configs/tax_rules_{год}.yaml` с полем `method: max_rate`
