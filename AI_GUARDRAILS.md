

# 🤖 AI Guardrails — Правила для ИИ-слоя

Этот документ описывает, КАК должен работать ИИ в проекте. Главная задача: ИИ объясняет результаты расчётов, но НЕ генерирует числа и НЕ даёт советов.

---

## 🎯 Назначение AI-слоя

ИИ в «ФинПомощнике» — это **переводчик с языка цифр на язык людей**. Он:
- ✅ Объясняет, почему платёж по кредиту именно такой
- ✅ Рассказывает, как инфляция влияет на вклад
- ✅ Предлагает следующие шаги (посмотреть сценарий, сравнить варианты)
-  НЕ генерирует числа самостоятельно
- ❌ НЕ даёт советов "купить/продать"
- ❌ НЕ заменяет математическое ядро

**Почему это важно**: LLM может ошибаться. Если она "придумает" цифру — это будет финансовая ошибка с реальными последствиями.

---

## 📥 Что получает AI (входные данные)

AI получает ТОЛЬКО три блока информации:

### 1. `result_json` — результаты расчётов из ядра
Это числа, которые уже посчитало математическое ядро. Пример:
```json
{
  "monthly_payment": "47073.46",
  "total_payment": "1129763.04",
  "overpayment": "129763.04",
  "psk": "0.185",
  "nominal_rate": "0.12"
}
```

**Важно**: AI НЕ должен пересчитывать эти числа. Он должен использовать их как есть.

### 2. `context_rules` — контекст правил
Это правила, которые влияют на расчёты. Пример:
```json
{
  "jurisdiction": "RU",
  "tax_year": 2026,
  "deposit_threshold": 160000,
  "safe_payment_ratio": 0.35
}
```

### 3. `user_profile` — профиль пользователя
Это информация о пользователе для персонализации объяснений. Пример:
```json
{
  "segment": "freelancer",
  "goal": "safety_cushion",
  "risk_tolerance": "low",
  "explanation_depth": "brief"
}
```

---

## 📤 Что должен делать AI (задача)

AI должен:
1. **Перевести цифры в понятный текст** — объяснить, что означают числа из `result_json`
2. **Добавить контекст** — почему так получилось (используя `context_rules`)
3. **Предложить следующий шаг** — что пользователь может сделать дальше
4. **Добавить дисклеймер** — обязательное предупреждение в конце

---

## 📝 Паттерн ответа (ОБЯЗАТЕЛЬНО)

Каждый ответ AI должен следовать структуре из 4 шагов:

### Шаг 1: Observation (Наблюдение)
Констатация факта на основе цифр из `result_json`.

**Примеры**:
- ✅ "Ваш ежемесячный платёж по кредиту составит 47 073₽"
- ✅ "Реальная доходность вклада после инфляции — 2.29%"
- ❌ "Вы будете платить много" (субъективно, без цифр)

### Шаг 2: Possible cause (Возможная причина)
Объяснение, почему так получилось, со ссылкой на формулу или правило.

**Примеры**:
- ✅ "Это связано с тем, что ПСК включает страховку и комиссии, поэтому реальные расходы выше номинальной ставки 12%"
- ✅ "Инфляция 8% 'съедает' большую часть номинальной доходности 10%"
- ❌ "Банк вас обманывает" (эмоционально, без оснований)

### Шаг 3: Question (Вопрос)
Предложение следующего шага пользователю.

**Примеры**:
- ✅ "Хотите посмотреть, как изменится платёж при досрочном погашении +5 000₽/мес?"
- ✅ "Хотите сравнить с альтернативными вариантами вкладов?"
- ❌ "Вам нужно срочно гасить кредит" (давление, совет)

### Шаг 4: Disclaimer (Дисклеймер — ОБЯЗАТЕЛЬНО)
Всегда в конце каждого ответа:
```
💡 Примерная аналитика. Проверьте расчёты. Не является инвестиционной рекомендацией.
```

---

## 🚫 Что ЗАПРЕЩЕНО

| Запрет | Почему | Пример нарушения |
|--------|--------|------------------|
| Генерировать числа | Может ошибиться, сломает детерминизм | "Вы сэкономите ~15 000₽" |
| Давать советы "купить/продать" | Нарушает финрегулирование | "Рекомендую купить ОФЗ" |
| Заменять ядро расчётов | Дублирование логики, рассинхрон | "Налог составит 5200₽" (без запроса к ядру) |
| Хранить PII в логах | 152-ФЗ violation | `logger.info(f"User {email}...")` |
| Прогнозировать рынок | Невозможно детерминированно | "Курс вырастет на 15%" |
| Использовать `float` | Потеря точности | `float(result_json["psk"])` |

## 🔍 Валидация вывода

Перед отправкой ответа пользователю, AI должен проверить свой ответ:

### Проверка 1: Нет ли сгенерированных чисел
Если ответ содержит числа, которых нет в `result_json` → **reject**

**Почему наивное сравнение строк не работает**: AI часто переформатирует числа
для пользователя — `0.185` → «18.5%», `0.0229` → «2.29%», `47073.46` → «47 073.46».
Прямое сравнение строк отклонит **любой** правильный ответ. Нужно сравнивать
**числовые значения** с допустимым множеством, порождённым из `result_json`.

**Как проверить**: сравнение числовых значений с допуском и нормализацией
```python
import re
from decimal import Decimal

def _normalize_number(s: str) -> Decimal:
    """Убирает пробелы/неразрывные пробелы, приводим запятую к точке."""
    s = s.replace("\xa0", "").replace(" ", "").replace(",", ".")
    return Decimal(s)

def _allowed_numbers(result_json: dict) -> set[Decimal]:
    """Все числа из result_json + производные (×100 для процентов, округлённые)."""
    allowed = set()
    for v in result_json.values():
        if not isinstance(v, (str, int, float)):
            continue
        try:
            base = Decimal(str(v))
        except Exception:
            continue
        allowed.add(base)
        # Если похоже на долю (|x| < 1), добавляем ×100 (процентное представление)
        if abs(base) < 1:
            allowed.add(base * 100)
        # Целая часть и копейки отдельно (например 47073 и 47073.46)
        allowed.add(base.quantize(Decimal("1")))  # округление до целого
    return allowed

def has_generated_numbers(response: str, result_json: dict, tol: Decimal = Decimal("0.01")) -> bool:
    """True, если в ответе есть число, не объяснимое данными result_json."""
    allowed = _allowed_numbers(result_json)
    # Ищем числа: 47 073.46 | 18.5% | 0.185 | 1 000 000
    for raw in re.findall(r"\d[\d\s.,]*\d|\d", response):
        try:
            n = _normalize_number(raw)
        except Exception:
            continue
        if any(abs(n - a) <= tol for a in allowed):
            continue
        return True  # найдено число, которого нет в данных
    return False
```

**Замечание о точности**: копейки всегда проверяем `quantize(Decimal("0.01"))`,
чтобы «47 073.4» не считалось сгенерированным при эталоне «47 073.46». При сомнениях —
лучше более мягкий допуск, чем false positive (отклонение правильного ответа).

### Проверка 2: Есть ли дисклеймер
Если в ответе нет дисклеймера → **reject**

**Как проверить**: поиск ключевого текста
```python
def has_disclaimer(response: str) -> bool:
    """Проверяет, есть ли в ответе обязательный дисклеймер"""
    return "💡" in response and "рекомендаци" in response.lower()
```

### Проверка 3: Нет ли советов "купить/продать"
Если в ответе есть запрещённые слова → **reject**

**Как проверить**: поиск запрещённых фраз
```python
def has_forbidden_advice(response: str) -> bool:
    """Проверяет, есть ли в ответе запрещённые советы"""
    forbidden_phrases = [
        "рекомендую купить",
        "советую продать",
        "срочно инвестируй",
        "покупайте",
        "продавайте"
    ]
    response_lower = response.lower()
    return any(phrase in response_lower for phrase in forbidden_phrases)
```

## 🔄 Fallback при нарушении

Если валидация не прошла, AI должен вернуть **шаблонный ответ** вместо сгенерированного:

```python
def fallback_template(result_json: dict, context_rules: dict) -> str:
    """Возвращает шаблонный ответ при нарушении guardrails"""
    
    # Извлекаем ключевые метрики
    metrics = []
    for key, value in result_json.items():
        if isinstance(value, (str, int, float)):
            metrics.append(f"- {key}: {value}")
    
    metrics_text = "\n".join(metrics) if metrics else "Нет данных"
    
    return f"""
📊 Результат расчёта:

{metrics_text}

💡 Примерная аналитика. Проверьте расчёты. Не является инвестиционной рекомендацией.
"""
```

**Пример fallback-ответа**:
```
📊 Результат расчёта:

- monthly_payment: 47073.46
- total_payment: 1129763.04
- overpayment: 129763.04
- psk: 0.185

💡 Примерная аналитика. Проверьте расчёты. Не является инвестиционной рекомендацией.
```

## 🎨 UI-требования

Рядом с каждым объяснением от AI должны быть:

### 1. Индикатор "Объяснено ИИ"
Визуальная пометка, что текст сгенерирован AI:
```
🤖 Объяснение от ИИ
```

### 2. Кнопка "Показать формулу"
При клике раскрывается блок с математической основой:
```
📐 Математическая основа:
Формула: A = P × [i×(1+i)^n] / [(1+i)^n − 1]
Источник: Копнова Г.П. "Финансовая математика", Гл. 4.2.2
Переменные:
  - P = 1 000 000₽ (сумма кредита)
  - i = 0.01 (месячная ставка = 12% / 12)
  - n = 24 (срок в месяцах)
Результат: A = 47 073.46₽
```

### 3. Кнопка "Пожаловаться на ответ"
Для сбора feedback и улучшения модели:
```
[👍 Полезно] [👎 Неточно] [ Пожаловаться]
```

## ✅ Примеры правильных ответов

### Пример 1: Объяснение ПСК кредита

**Вход**:
```json
{
  "psk": "0.185",
  "nominal_rate": "0.12",
  "insurance": "50000"
}
```

**Правильный ответ**:
```
📊 Полная стоимость кредита (ПСК) составляет 18.5%.

Это выше номинальной ставки 12%, потому что ПСК включает все обязательные 
платежи: страховку (50 000₽) и комиссии банка. Методика расчёта ПСК 
утверждена ЦБ РФ (Указание 5832-У).

Хотите посмотреть, как изменится ПСК без страховки?

💡 Примерная аналитика. Проверьте расчёты. Не является инвестиционной рекомендацией.
```

### Пример 2: Объяснение реальной доходности вклада

**Вход**:
```json
{
  "nominal_yield": "0.10",
  "inflation": "0.08",
  "real_yield": "0.0229"
}
```

**Правильный ответ**:
```
📊 Реальная доходность вашего вклада после инфляции — 2.29%.

Номинально вклад приносит 10% годовых, но инфляция 8% "съедает" 
большую часть. Формула Фишера: r_real = (1 + r_nom) / (1 + π) − 1.

Хотите сравнить с альтернативными инструментами (ОФЗ, акции)?

💡 Примерная аналитика. Проверьте расчёты. Не является инвестиционной рекомендацией.
```

## ❌ Примеры неправильных ответов

### Нарушение 1: Сгенерированное число

**Неправильно**:
```
📊 Ваш вклад принесёт вам примерно 110 000₽ через год.
```

**Почему неправильно**: Число "110 000₽" не было в `result_json`. AI его "придумал".

**Как исправить**: Использовать только числа из `result_json`.

---

### Нарушение 2: Инвестиционный совет

**Неправильно**:
```
📊 Реальная доходность 2.29%. Рекомендую рассмотреть также акции — они могут дать большую доходность.
```

**Почему неправильно**: "Рекомендую рассмотреть акции" — это инвестиционный совет.

**Как исправить**: "Хотите сравнить с альтернативными инструментами?"

---

### Нарушение 3: Нет дисклеймера

**Неправильно**:
```
📊 Полная стоимость кредита (ПСК) составляет 18.5%. Это выше номинальной ставки 12%, потому что ПСК включает страховку и комиссии.
```

**Почему неправильно**: Отсутствует обязательный дисклеймер.

**Как исправить**: Добавить в конец: "💡 Примерная аналитика. Проверьте расчёты. Не является инвестиционной рекомендацией."

## ⚠️ Edge-cases

| Ситуация | Обработка |
|----------|-----------|
| `result_json` пустой | Вернуть fallback-шаблон без объяснений |
| Пользователь спрашивает "что купить?" | Ответ: "Я не даю инвестиционных советов. Могу объяснить расчёты." |
| LLM не отвечает (timeout) | Вернуть fallback-шаблон, залогировать инцидент |
| Пользователь просит прогноз рынка | Ответ: "Прогнозы рынка невозможны детерминированно. Могу показать сценарий 'что если'." |
| Противоречие в `context_rules` | Использовать приоритет: РФ > общие правила, логировать warning |

## 📊 Логирование

Все ответы AI должны логироваться для аудита:

**Что логировать**:
- `calc_id` — идентификатор расчёта
- `prompt_hash` — хэш промпта (для отслеживания дубликатов)
- `response_hash` — хэш ответа
- `is_valid` — прошла ли валидация
- `validation_error` — ошибка валидации (если была)

**Что НЕ логировать**:
- Сырые данные пользователя (PII)
- Токены доступа
- Пароли

**Пример лога**:
```json
{
  "calc_id": "hash_123",
  "prompt_hash": "sha256_abc...",
  "response_hash": "sha256_def...",
  "is_valid": true,
  "validation_error": null,
  "timestamp": "2024-05-15T15:00:00Z"
}
```

## ✅ Чек-лист готовности AI-слоя

При реализации AI-слоя проверить:
- [ ] AI получает ТОЛЬКО `result_json`, `context_rules`, `user_profile`
- [ ] AI НЕ генерирует числа самостоятельно
- [ ] AI НЕ даёт советов "купить/продать"
- [ ] AI следует паттерну: Observation → Cause → Question → Disclaimer
